# terraci update

Проверка доступных обновлений для провайдеров и модулей Terraform в проекте.

## Синтаксис

```bash
terraci update [flags]
```

## Описание

Команда `update` сканирует Terraform-файлы в обнаруженных модулях, запрашивает Terraform Registry и сообщает о доступных обновлениях версий провайдеров и модулей.

По умолчанию работает в режиме только чтения — выводит отчёт о доступных обновлениях без изменения файлов. Используйте `--write` для записи обновлённых версий обратно в `.tf` файлы.

Команда не завершается с ошибкой при наличии доступных обновлений — ненулевой код возврата означает ошибку выполнения (сбой парсинга, ошибка реестра или ошибка записи).

## Флаги

| Флаг | Сокр. | Тип | По умолчанию | Описание |
|------|-------|-----|--------------|----------|
| `--target` | `-t` | string | `all` | Что проверять: `modules`, `providers`, `all` |
| `--bump` | `-b` | string | `minor` | Уровень версионирования: `patch`, `minor`, `major` |
| `--write` | `-w` | bool | `false` | Записать обновлённые версии в `.tf` файлы |
| `--module` | `-m` | string | | Проверить только указанный модуль |
| `--output` | `-o` | string | `text` | Формат вывода: `text`, `json` |

## Примеры

```bash
# Проверить все зависимости (провайдеры и модули)
terraci update

# Проверить только провайдеры
terraci update --target providers

# Проверить только модули с уровнем patch-обновлений
terraci update --target modules --bump patch

# Записать обновлённые версии в .tf файлы
terraci update --write

# Проверить конкретный модуль
terraci update --module platform/prod/eu-central-1/vpc

# JSON вывод
terraci update --output json
```

## Уровни версионирования

Флаг `--bump` определяет, какие обновления считаются «доступными»:

| Значение | Описание | Пример: текущая `~> 5.1` |
|----------|----------|--------------------------|
| `patch` | Только patch-обновления | `~> 5.1` → `~> 5.1` (ограничение ~>) |
| `minor` | Minor и patch-обновления (по умолчанию) | `~> 5.1` → `~> 5.80` |
| `major` | Major, minor и patch-обновления | `~> 5.1` → `~> 6.0` |

## Обработка ограничений версий

TerraCi читает и записывает ограничения версий Terraform, сохраняя их стиль.

### Распознавание ограничений

Ограничение `~> 5.80` распознаётся как базовая версия `5.80`. TerraCi использует базовую версию для определения применимых обновлений с учётом флага `--bump`.

### Поведение при записи (`--write`)

При записи стиль ограничения сохраняется:

| Текущее ограничение | Последняя версия | Результат |
|---------------------|-----------------|-----------|
| `~> 5.0` | `5.82.0` | `~> 5.82` |
| `~> 5.0.1` | `5.1.3` | `~> 5.1.3` |
| `>= 1.0` | `2.0.0` | `>= 2.0` |

Правило: если текущее ограничение задаёт точность до patch-версии (например, `~> 5.0.1`), результирующее ограничение тоже будет с patch-точностью. Если до minor (например, `~> 5.0`) — результат будет minor-точности.

### Поддерживаемые операторы

Поддерживаются следующие операторы ограничений:

- `~>` — pessimistic constraint (тильда, наиболее распространённый)
- `>=` — больше или равно
- `<=` — меньше или равно
- `>` — строго больше
- `<` — строго меньше
- `=` — точная версия
- `!=` — исключение версии

### Составные ограничения

Составные ограничения вида `">= 1.0, < 2.0"` также поддерживаются. При записи обновляется только тот оператор, который указывает нижнюю границу (например, `>=`), верхняя граница (`< 2.0`) сохраняется.

## Режим записи

При использовании `--write` TerraCi обновляет ограничения версий непосредственно в `.tf` файлах:

```bash
terraci update --write
```

Поведение при записи:
- Обновляются только файлы с доступными обновлениями
- Ограничения версий обновляются в соответствии с заданным уровнем `--bump`
- Оригинальное форматирование файла сохраняется
- Неподдерживаемые форматы ограничений пропускаются

## Требования

- `plugins.update.enabled: true` в `.terraci.yaml`
- Доступ к интернету для запросов к Terraform Registry

## Коды завершения

| Код | Описание |
|-----|----------|
| 0 | Сканирование завершено успешно (даже при наличии доступных обновлений) |
| ненулевой | Ошибка парсинга, запроса к реестру или записи файлов |

## Конфигурация

Настройте поведение по умолчанию через `.terraci.yaml`:

```yaml
plugins:
  update:
    enabled: true
    target: all       # all, modules, providers
    bump: minor       # patch, minor, major
    ignore:
      - registry.terraform.io/hashicorp/null
      - registry.terraform.io/hashicorp/random
    pipeline: false   # Добавить джоб проверки в CI пайплайн
```

Флаги командной строки имеют приоритет над настройками конфигурационного файла.

## Интеграция с CI пайплайном

При `plugins.update.pipeline: true` TerraCi добавляет джоб `dependency-update-check` в начало пайплайна (фаза до plan). Джоб помечается как `allow_failure: true`, поэтому наличие доступных обновлений не блокирует деплой.

```yaml
plugins:
  update:
    enabled: true
    pipeline: true
```
